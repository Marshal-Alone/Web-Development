SQL PART-1 : Notes

1. Creating Databases

  • Command: 
    CREATE DATABASE college;
    Creates a database named college.


  • Conditional Creation: 
    CREATE DATABASE IF NOT EXISTS instagram;
    Ensures the database instagram is created only if it doesn't already exist.


  • Switching to a Database: 
    USE instagram;
    Sets the current database context to instagram.


2. Creating Tables with Constraints

  • User Table: 
    CREATE TABLE user (
    Id INT AUTO_INCREMENT PRIMARY KEY,
    Name VARCHAR(50),
    Email VARCHAR(50) UNIQUE,
    Followers INT,
    Following INT,
    Age INT,
    CONSTRAINT CHECK (age >= 13)
);
    • Columns:
      • Id: Auto-incrementing primary key.
      • Name: Stores names up to 50 characters.
      • Email: Unique constraint ensures no duplicate emails.
      • Followers and Following: Integers to track social stats.
      • Age: Must be 13 or older (enforced by CHECK).


  • Post Table: 
    CREATE TABLE post (
    id INT PRIMARY KEY,
    content VARCHAR(100),
    user_id INT,
    FOREIGN KEY (user_id) REFERENCES user (id)
);


    • Columns:
      • id: Primary key.
      • content: Text content up to 100 characters.
      • user_id: Foreign key referencing user (id).


3. Inserting Data

  • Adding Users: 
    INSERT INTO user (Id, Name, Email, Followers, Following, Age)
VALUES
(1, "Adam", "adam@gmail.com", 123, 145, 15),
(2, "Eve", "eve@gmail.com", 200, 200, 25);


  • Inserting Without Specifying Columns: 
    INSERT INTO user 
VALUES

4. Querying Data

  • Retrieve All Columns: 
    SELECT * FROM user;
    Fetches all rows and columns from the user table.


  • Specific Columns: 
    SELECT id, name, email FROM user;
    Returns only id, name, and email.


  • Filter Unique Values: 
    SELECT DISTINCT age FROM user;
    Retrieves distinct ages, avoiding duplicates.


Key Concepts Reinforced

  • Primary Keys: Ensure each row in a table is uniquely identifiable.


  • Foreign Keys: Link tables to establish relationships.


  • Constraints:
    • UNIQUE: Prevents duplicate entries.
    • CHECK: Validates data against a condition.


  • Queries:
    • SELECT: Fetch data.
    • DISTINCT: Remove duplicates.
